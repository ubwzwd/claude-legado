<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-supplied source file path -> file system | Path traversal risk when copying source to SOURCES_DIR |
| Book source `header` field -> httpx headers | Malformed JSON or unexpected types in header field |
| External HTTP response -> decode_response() | Malicious or unexpected encoding in response bytes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | _use_source() file copy | mitigate | Use `Path(args[0]).name` (basename only) as destination filename in SOURCES_DIR; never use full user-supplied path as dest component |
| T-03-02 | Tampering | parse_source_headers() | mitigate | Wrap `json.loads()` in try/except JSONDecodeError, return empty dict on failure; validate result is dict type |
| T-03-03 | Denial of Service | fetch() timeout | mitigate | httpx.Client timeout=10.0 default; httpx max_redirects=20 default prevents infinite redirect loops |
| T-03-04 | Spoofing | SSRF via book source URLs | accept | Local CLI tool run by user who chose the book source; no elevated privilege. Phase 5 hardening scope. |
| T-03-05 | Information Disclosure | decode_response errors='replace' | accept | Replacement characters are visible to user; no silent data corruption risk for a novel reader |
</threat_model>
