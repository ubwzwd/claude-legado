# Phase 04-04: Content Flow

<objective>
Implement `/novel` and `/novel next` to fetch chapter content, stream it via existing display logic, and track reading position.
</objective>

<threat_model>
- Input validation: None from user, all urls derive from source rules. Safe.
- Race conditions: If user interrupts mid-stream, progress could be lost if not saved first. Mitigation: D-10 mandates saving progress before stream.
</threat_model>

<requirements>
- FLOW-04: Content flow (fetch, follow rules, extract and stream text).
- DISP-01: Connect extracted text to `display.stream_chapter`.
- STATE-02/D-10: Save progress before streaming.
</requirements>

<action>
Update `_stream_current()` and `_dispatch_next_prev()` in `src/novel/commands.py`.
1. Retrieve active book from state/shelf, and its current chapter URL/index.
2. If `next` passed, increment index.
3. Save state to `state.json` (D-10) with the new position.
4. Use `novel.http.follow_content_pages` to fetch the chapter page(s) accounting for `nextContentUrl` pagination.
5. Parse the collected HTML via `novel.rules.evaluate` against `ruleContent`.
6. Extract title (using `ruleContent_title` or default to TOC title).
7. Pass the extracted string to `display.stream_chapter()`. 
Update `state.py` to store current `chapter_idx` against the shelf book.
</action>
