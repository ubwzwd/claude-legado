# Phase 04-01: Search Flow

<objective>
Implement `/novel search <query>` to fetch real search results and display them using the rule engine.
</objective>

<threat_model>
- Injection: Search query passed directly to URL template - must URL-encode properly.
- Denial of Service: Malformed regex in ruleSearch causing catastrophic backtracking. Mitigation: rely on phase 2 rule engine exceptions and catch RuleError.
</threat_model>

<requirements>
- FLOW-01: Search flow: build URL, fetch, regex rules, return list
- STATE-01: Persist state to `state.json` (for short-lived cache)
</requirements>

<tasks>

<read_first>

</read_first>
<action>
Add `save_search_cache(books: list[dict])` and `load_search_cache() -> list[dict]` to `src/novel/state.py`.

`books` will be a list of dicts with keys like `name`, `author`, `bookUrl`, `coverUrl`, `origin_source_id`.
</action>

<task id="04-01-02" name="Wire up /novel search command">
<read_first>
- src/novel/commands.py
- .planning/phases/04-read-pipeline/04-CONTEXT.md
</read_first>
<action>
Update `_search_books(console, args)` in `src/novel/commands.py` to:
1. Load the active source using `state.load_state()`.
2. Grab the `searchUrl` from the source config, replacing `{{key}}` with the url-encoded search query (`args[0]`). If `{{page}}` exists, replace with `1`.
3. Call `novel.http.fetch(url, source_config)`.
4. Run `novel.rules.evaluate_list(ruleSearch, content)` to extract book items.
5. For each item string, `novel.rules.evaluate` sub-rules (`ruleSearchName`, `ruleSearchAuthor`, `ruleSearchNote` etc.) to build a dictionary.
6. Print the results in format `[1] Title - Author` (D-08).
7. Save the result list using `state.save_search_cache()`.
If a sub-rule fails or is missing, use default empty string or fallback handling.
</action>

- `pytest tests/test_04_read_pipeline.py -k test_search_flow` passing
</acceptance_criteria>
</task>
