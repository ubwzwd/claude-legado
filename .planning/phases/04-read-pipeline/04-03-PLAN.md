# Phase 04-03: TOC Flow

<objective>
Implement `/novel toc <page>` to follow multi-page TOCs and list ordered chapters.
</objective>

<threat_model>
- Memory exhaustion: Following infinite TOC pages if nextTocUrl is a self-referencing loop. Mitigation: the httpx hook limits to reasonable bound or check previously visited pages.
</threat_model>

<requirements>
- FLOW-03: Fetch TOC pages iteratively, parse `ruleToc`, display.
</requirements>

<tasks>

<task id="04-03-01" name="Implement /novel toc">
<read_first>
- src/novel/commands.py
- .planning/phases/04-read-pipeline/04-CONTEXT.md
</read_first>
<action>
Update `_show_toc` in `commands.py` to:
1. Identify active book from `state.json` and `shelf.json`.
2. Extract the book's `tocUrl` (or `bookUrl` if `tocUrl` absent).
3. Use `novel.http.follow_toc_pages` to fetch all TOC pages. Give it the `nextTocUrl` rule and the `apply_url_template` helper.
4. For each page HTML, run `ruleToc` (returns list of items).
5. For each item, extract `ruleTocName` and `ruleTocUrl`.
6. Concatenate all chapter dicts in order.
7. Paginate the output to display e.g., 50 chapters depending on shell args (D-04).
8. Cache the TOC array in `shelf.json` against the book so `/novel next` knows the chapter URLs. Update `save_shelf`.
</action>
<acceptance_criteria>
- `cat src/novel/commands.py` contains `follow_toc_pages`
- `pytest tests/test_04_read_pipeline.py -k test_toc_flow` passing
</acceptance_criteria>
</task>

</tasks>
