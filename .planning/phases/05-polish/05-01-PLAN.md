# Plan 05-01: Error Handling Hardening

<objective>
Ensure all exceptions during source loading, network fetching, and rule evaluation are caught and streamed in the Claude "thinking" format, preventing stack traces from leaking to the terminal.
</objective>

<requirements>
- Phase-5-Hardening
</requirements>

<must_haves>
- Graceful degradation on network timeout or malformed sources.
- stream_error() must mimic Claude's visual persona (italic dim).
- Remove fallback to FAKE_BOOK on start, showing error instead.
</must_haves>
